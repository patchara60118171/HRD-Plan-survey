## 📋 **Project Context**

### **🏗️ Project Architecture**
```
NIDA Well-being Survey/
├── 🌐 Frontend (HTML/JS/CSS)
│   ├── admin.html (7-page SPA, 7,418 lines)
│   ├── ch1.html (5-step form)
│   └── index.html (general survey)
│
├── 🗄️ Backend (Supabase)
│   ├── PostgreSQL Database
│   ├── Authentication System
│   └── File Storage
│
├── 🧪 Testing & Automation
│   ├── Playwright Scripts
│   └── Form Testing
│
└── 📚 Documentation
    ├── Setup Guides
    ├── User Manuals
    └── API Documentation
```

### **🔧 Key Technologies**
- **Frontend:** HTML5, Tailwind CSS, Vanilla JavaScript
- **Backend:** Supabase (PostgreSQL + Auth + Storage)
- **Deployment:** Vercel + CI/CD
- **Testing:** Playwright Automation
- **Security:** RLS, RBAC, Anti-replay

### **📊 Database Tables**
- `organizations` - ข้อมูลหน่วยงาน
- `survey_forms` - รีจิสทรีฟอร์ม
- `org_form_links` - จัดการลิงก์
- `wellbeing_responses` - คำตอบแบบสอบถามทั่วไป
- `hrd_ch1_responses` - คำตอบ HRD บทที่ 1
- `admin_user_roles` - จัดการสิทธิ์ผู้ใช้
